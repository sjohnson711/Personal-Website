================================================================================
DEPLOYMENT & TROUBLESHOOTING GUIDE FOR MONOREPO PROJECTS
================================================================================

This guide documents how to deploy a full-stack monorepo (frontend + backend + database)
to Vercel + Railway + Neon, and how to troubleshoot common issues that arose in this project.

PART 1: PROJECT STRUCTURE FOR MONOREPO DEPLOYMENTS
================================================================================

DO THIS FIRST - Organize as a monorepo with separate frontend/backend:

```
blog-site/
├── frontend/          ← React/Vite app (deploys to Vercel)
│   ├── src/
│   ├── package.json   ← MUST have: react, vite, @vitejs/plugin-react
│   ├── vercel.json    ← SPA rewrites (not needed at root)
│   └── vite.config.ts
├── backend/           ← Express server (deploys to Railway)
│   ├── src/
│   ├── package.json   ← MUST have: express, prisma, typescript
│   ├── prisma/
│   │   ├── schema.prisma
│   │   ├── migrations/
│   │   └── seed.ts
│   └── .env
├── package.json       ← ROOT: minimal, declares workspaces
└── vercel.json        ← ROOT: tells Vercel how to build frontend
```

KEY: Root package.json should be minimal with workspaces, NOT a full config.

PART 2: CRITICAL DEPENDENCIES BY LAYER
================================================================================

FRONTEND (Vite + React):

Required for Build:
  ✓ @vitejs/plugin-react        ← Handles JSX transformation
  ✓ vite                         ← Bundler (replacing webpack)
  ✓ typescript                   ← Type checking (tsc runs before vite)
  ✓ @tailwindcss/vite           ← Tailwind CSS v4 Vite plugin
  ✓ @tailwindcss/postcss        ← Required for PostCSS integration in v4
  ✓ tailwindcss                  ← Utility CSS framework

Required for Runtime:
  ✓ react & react-dom           ← UI library
  ✓ react-router-dom            ← Client-side routing (required for SPA)
  ✓ marked                       ← Markdown rendering
  ✓ slugify                      ← URL-safe slug generation

Common mistakes:
  ✗ Missing react-router-dom    → Error: "Cannot find module 'react-router-dom'"
  ✗ Missing @tailwindcss/postcss → Error: "Cannot find module '@tailwindcss/postcss'"
  ✗ Using @tailwindcss/vite v3  → Incompatible with Tailwind v4

BACKEND (Express + Prisma):

Required:
  ✓ express                      ← Web server
  ✓ @prisma/client              ← ORM (generated to src/generated/prisma/)
  ✓ @prisma/adapter-pg          ← PostgreSQL driver adapter (Prisma v7)
  ✓ pg                           ← PostgreSQL client (peer dependency)
  ✓ prisma                       ← CLI (dev dependency)
  ✓ typescript                   ← Type checking
  ✓ tsx                          ← TypeScript executor for seed.ts

Important notes:
  - Prisma v7 uses driver adapters; must have @prisma/adapter-pg + pg
  - prisma.schema.ts has NO url field — connection comes from PrismaPg(DATABASE_URL)
  - Prisma client generated to src/generated/prisma (check schema.prisma)

DATABASE (Neon PostgreSQL):

Setup:
  1. Create project at neon.tech
  2. Copy full connection string with sslmode=require & channel_binding=require
  3. Store as DATABASE_URL environment variable

Migration & Seeding:
  # Run migrations (apply schema)
  DATABASE_URL="<neon-url>" npx prisma migrate deploy

  # Seed data (create admin, sample articles)
  DATABASE_URL="<neon-url>" ADMIN_PASSWORD="<password>" npx tsx prisma/seed.ts

PART 3: STEP-BY-STEP DEPLOYMENT FLOW
================================================================================

STEP 1: Create Root package.json with Workspaces
────────────────────────────────────────────────

File: package.json
```json
{
  "name": "your-project",
  "version": "1.0.0",
  "private": true,
  "workspaces": ["frontend", "backend"]
}
```

Why: Tells npm to install dependencies in both frontend/ and backend/
     Single npm install handles all projects.

STEP 2: Create Root vercel.json
────────────────────────────────

File: vercel.json
```json
{
  "buildCommand": "npm install && cd frontend && npm run build",
  "outputDirectory": "frontend/dist",
  "env": {
    "VITE_API_URL": "@vite-api-url"
  }
}
```

Why: Tells Vercel to:
     - Run npm install (installs all workspace dependencies)
     - cd into frontend and run vite build
     - Serve from frontend/dist
     - Use VITE_API_URL env var (set in Vercel dashboard)

STEP 3: Neon Setup
──────────────────

1. Sign up at https://neon.tech
2. Create project
3. Copy connection string (contains password — keep secret!)
4. Save for Railway setup (Step 5)

STEP 4: Railway Backend Setup
──────────────────────────────

1. Sign up at https://railway.app
2. Connect GitHub account
3. Create project → "Deploy from GitHub"
4. Select your repo
5. Set root directory: /backend
6. Add environment variables (in Railway dashboard):
   - DATABASE_URL (from Neon)
   - JWT_SECRET (generate one or use existing)
   - FRONTEND_URL (set after Vercel domain is ready)
   - SITE_URL (same as FRONTEND_URL)
   - RESEND_API_KEY (if using email)
   - FROM_EMAIL (if using email)
   - ADMIN_PASSWORD (for seeding)

7. Railway auto-deploys on GitHub push
8. Copy Railway public domain (e.g., https://your-project.railway.app)

STEP 5: Vercel Frontend Setup
──────────────────────────────

1. Sign up at https://vercel.com
2. Import your GitHub repo
3. DO NOT select a framework — it will auto-detect
4. Set Root Directory: . (root, not frontend/)
5. Set Build Command: npm install && cd frontend && npm run build
6. Set Output Directory: frontend/dist
7. Add environment variables:
   - VITE_API_URL = https://your-railway-domain.railway.app
8. Deploy
9. Copy Vercel domain (e.g., https://your-project.vercel.app)

STEP 6: Update Railway with Vercel Domain
──────────────────────────────────────────

1. Go back to Railway dashboard
2. Update environment variables:
   - FRONTEND_URL = https://your-vercel-domain.vercel.app
   - SITE_URL = https://your-vercel-domain.vercel.app
3. Redeploy
4. Test: https://your-vercel-domain.vercel.app should load

STEP 7: Sync Local Data to Production
──────────────────────────────────────

Export local articles:
  pg_dump --username=<user> --dbname=<db> \
    --table='"Article"' --data-only --column-inserts \
    > articles.sql

Import to Neon:
  psql "<neon-connection-string>" -f articles.sql

Seed production database:
  DATABASE_URL="<neon-url>" ADMIN_PASSWORD="<password>" npx tsx prisma/seed.ts

PART 4: COMMON ERRORS & HOW TO FIX THEM
================================================================================

ERROR 1: "Cannot find module 'react-router-dom'"
────────────────────────────────────────────────

Cause: npm install didn't run in frontend/ directory
Solution:
  1. Check frontend/package.json has react-router-dom listed
  2. Check Root vercel.json has: buildCommand: "npm install && cd frontend && npm run build"
  3. Trigger new Vercel deployment

ERROR 2: "Cannot find module '@tailwindcss/postcss'"
───────────────────────────────────────────────────

Cause: Tailwind v4 requires @tailwindcss/postcss in devDependencies
Solution:
  1. Add to frontend/package.json devDependencies:
     "@tailwindcss/postcss": "^4"
  2. Commit and push
  3. Trigger new Vercel deployment

ERROR 3: "Command 'npm run build' exited with 254: Cannot read package.json"
────────────────────────────────────────────────────────────────────────────

Cause: Vercel looking for package.json in root, but file was deleted
Solution:
  1. Create root package.json with workspaces (see STEP 1 above)
  2. Create root vercel.json with correct buildCommand (see STEP 2 above)
  3. Commit and push
  4. Trigger new deployment

ERROR 4: "Building with Next.js instead of Vite"
─────────────────────────────────────────────────

Cause: Root package.json has "build": "next build"
Solution:
  1. Delete root package.json
  2. Create minimal root package.json with workspaces only
  3. Ensure vercel.json buildCommand is: "npm install && cd frontend && npm run build"
  4. Commit, push, redeploy

ERROR 5: "TS2774: This condition will always return true"
──────────────────────────────────────────────────────────

Cause: TypeScript thinks navigator.share is always defined (it's not, varies by browser)
Solution:
  1. Use optional chaining: navigator?.share
  2. Or add @ts-expect-error comment above the check
  3. Or use: typeof navigator.share === "function"

ERROR 6: "TS6133: Variable declared but never used"
─────────────────────────────────────────────────────

Cause: Dead code or variable no longer needed
Solution:
  1. Delete the unused variable
  2. If intentionally unused, prefix with _ (e.g., _encodedExcerpt)

PART 5: DEBUGGING CHECKLIST FOR BUILD FAILURES
================================================================================

When your Vercel build fails:

1. CHECK THE BUILD LOG (not terminal output):
   ⏳ Go to Vercel dashboard
   ⏳ Click "Deployments"
   ⏳ Click the failed deployment
   ⏳ Scroll to the bottom — the actual error is there
   ⏳ Copy the full error message (sometimes truncated in console)

2. IDENTIFY ERROR TYPE:
   ⏳ Module not found? → Missing dependency in package.json
   ⏳ TypeScript error (TS####)? → Type issue, check file causing error
   ⏳ Build tool error? → Check vite.config.ts or postcss.config
   ⏳ npm install failed? → Check root package.json and vercel.json

3. VERIFY FILE STRUCTURE:
   ⏳ Check root package.json exists and has workspaces
   ⏳ Check root vercel.json exists and has correct buildCommand
   ⏳ Check frontend/package.json has all required dependencies
   ⏳ Check frontend/vite.config.ts is correct

4. VERIFY ENVIRONMENT VARIABLES:
   ⏳ Go to Vercel dashboard → Settings → Environment Variables
   ⏳ Confirm VITE_API_URL is set to your Railway backend domain
   ⏳ Confirm all vars are set before deploying

5. LOCAL TESTING:
   ⏳ Run `npm install` locally to verify dependencies install
   ⏳ Run `npm run build` locally to verify build works
   ⏳ If it builds locally but fails on Vercel, issue is environment-specific
   ⏳ Check for hardcoded paths, absolute URLs, or missing env vars

6. COMMON QUICK FIXES:
   ⏳ Delete node_modules and package-lock.json, reinstall
   ⏳ Clear Vercel build cache and redeploy
   ⏳ Check git status — ensure changes are committed
   ⏳ Verify you pushed to the right branch (should be main)
